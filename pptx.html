1) Problema, objetivo y alcance

Problema operativo

  • Dependencia de expertos para construir SQL.
  • Tiempos altos para extraer insights y reportes reutilizables.
  • Dificultad para explotar manuales tecnicos con trazabilidad.
  • Necesidad de control corporativo y gobernanza.

Objetivo tecnico

  • Democratizar analitica con interfaz conversacional.
  • Reducir friccion entre pregunta de negocio y datos.
  • Estandarizar reporting HTML/PDF/CSV.
  • Integrar consulta documental con RAG contextual.

2) Capacidades funcionales

Chat de datos (Text-to-SQL)

  • Input por texto o audio.
  • Agente LLM genera SQL y valida formato.
  • Ejecucion SELECT-only y reintentos por sintaxis.
  • Tabla + metadatos tecnicos para supervisor.

Chat with Data

  • Respuesta breve en lenguaje natural desde dataframe.
  • Complementa SQL/tablas con narrativa ejecutiva.

Report Builder

  • Plantillas HTML + componentes visuales.
  • Recomendador de grafica con agente LLM.
  • Exportacion a PDF y CSV.

Manuals Chat (RAG)

  • Webhook n8n con API key y sessionId.
  • Parseo flexible de respuestas del workflow.
  • Entrada por voz para soporte tecnico.

3) Tecnologias usadas

Frontend y UX

Interfaz, layout y experiencia

• Python 3.11 + Streamlit • Reporting HTML con CSS Grid • Interfaz conversacional responsive

IA y datos

Modelos, agentes y base de datos

• LangChain + Azure OpenAI • Ollama como fallback local • PostgreSQL + sqlglot SELECT-only

Integracion y despliegue

Automatizacion y operación

• n8n para RAG y automatizacion • Azure Entra + Graph API • Docker Compose + Kubernetes
Python 3.11 Streamlit LangChain Azure OpenAI Ollama PostgreSQL sqlglot n8n Docker Kubernetes

8) Seguridad, gobierno y despliegue

Backend + IA

  • Python 3.11, Streamlit, Pydantic Settings.
  • LangChain con Azure Chat Completions y opcion Ollama.
  • Prompts con contexto y control de tokens (tiktoken).

Datos + seguridad

  • PostgreSQL y validacion SQL via sqlglot (SELECT-only).
  • SSO Azure Entra + autorizacion por grupos.
  • Secretos en env/configmap/secret fuera de codigo.

Operacion

  • Local/dev con Docker Compose.
  • Produccion en contenedor Streamlit + DB.
  • Kubernetes con Deployment/Service/Ingress/Config/Secret.
Azure Entra + Graph API sqlglot SELECT-only ConfigMap + Secret Docker + Kubernetes

9) Validacion funcional y experiencia de uso

Lo que ya valida el proyecto

  • Generacion SQL sobre lenguaje natural con validacion de seguridad.
  • Generacion de respuestas explicativas con Chat with Data.
  • Construccion de reportes tecnicos con exportables PDF/CSV.
  • Consulta contextual de manuales via flujo RAG n8n.

Valor para operaciones

Reduce dependencia de especialistas SQL y acelera tiempo de respuesta para consultas operativas y reportes reutilizables.

Incluye ademas control de acceso corporativo y gobernanza de datos.

10) Calidad del proyecto

Calidad tecnica

  • Tests para agentes, app y flujo RAG.
  • Script de validacion SQL con metricas AST/columnas/tiempos.
  • Arquitectura modular por capas para evolucion controlada.
test_agent.py test_app.py test_rag.py validate.py

Criterios de evaluacion tecnica

  • Seguridad: SSO, autorizacion por grupos y SQL en modo solo lectura.
  • Escalabilidad: despliegue portable en Docker/Kubernetes.
  • Mantenibilidad: separacion clara entre UI, control, IA e integraciones.
  • Usabilidad: experiencia conversacional con salida tecnica y ejecutiva.

11) Summary de la app

Que resuelve

Convierte preguntas en lenguaje natural en analitica, reportes y consulta documental gobernada.

Como lo hace

Combina Streamlit, agentes IA, validacion SQL, RAG con n8n, y exportacion de resultados reutilizables.

Valor final

Reduce dependencia de especialistas, acelera la toma de decisiones y aporta trazabilidad tecnica.

12) Posibles preguntas tecnicas

1. Como evitas que el LLM genere SQL inseguro?

Con validacion previa del SQL, modo SELECT-only y separacion entre generacion, verificacion y ejecucion.

2. Que ventaja aporta n8n en Manuals Chat?

Desacopla el flujo RAG, permite iterar rapido y mantiene el pipeline de retrieval y generacion más flexible.

3. Como se adapta la solucion a entorno corporativo?

Con autenticacion Entra, control por grupos, despliegue en contenedores y componentes preparados para Kubernetes.
