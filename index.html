1) Problema, objetivo y alcance

El problema real

  • Los centros de control de empresas de transporte (p. ej. ferroviario) registran datos operativos en tiempo real: trayectos, incidencias, estados por linea y estacion.
  • Las bases de datos son locales e internas; los datos se insertan en vivo mientras los trenes circulan a lo largo del dia, la semana o el año.
  • Los operarios no tienen conocimiento tecnico para escribir SQL, por lo que no pueden consultar esos datos sin depender de un especialista.
  • Acceder a manuales tecnicos del software del centro de control es lento y poco trazable.

Objetivo del proyecto

  • Proporcionar a operarios no tecnicos una interfaz conversacional en lenguaje natural para consultar datos operativos en tiempo real.
  • Permitir seleccionar esos datos y exportarlos como reportes tecnicos (HTML / PDF / CSV) sin intermediarios.
  • Ofrecer un chat con manuales del software del centro de control mediante RAG, para soporte tecnico rapido y contextualizado.
  • Todo esto sin requerir conocimiento SQL ni formacion tecnica avanzada por parte del usuario.

Nota: Este proyecto se ha desarrollado sobre una aplicacion base ya existente. El trabajo realizado comprende la extension, integracion y validacion tecnica de la plataforma, no su construccion desde cero.

2) Capacidades funcionales

1

Chat de datos (Text-to-SQL)

  • Entrada en lenguaje natural (texto o audio).
  • LLM genera y valida SQL de forma segura.
  • Ejecución SELECT-only en la base de datos.
  • Devuelve tabla con resultados + narrativa.
2

Chat with Data

  • Respuesta breve en lenguaje natural desde dataframe.
  • Complementa SQL/tablas con narrativa ejecutiva.
3

Report Builder

  • Templating visual con componentes HTML.
  • IA recomienda la mejor visualización.
  • Exportación a PDF, CSV y HTML.
4

Manuals Chat (RAG)

  • Chat sobre manuales del software.
  • Integración con n8n para RAG.
  • Entrada por voz para operarios.

8) Validacion funcional y experiencia de uso

Validacion funcional

  • Generacion SQL sobre lenguaje natural con validacion de seguridad.
  • Generacion de respuestas explicativas con Chat with Data.
  • Construccion de reportes tecnicos con exportables PDF/CSV.
  • Consulta contextual de manuales via flujo RAG n8n.

Experiencia de uso

  • Elimina la dependencia de especialistas SQL en el centro de control: cualquier operario puede consultar datos operativos en tiempo real usando lenguaje natural y obtener reportes sin intermediarios.
  • Incluye ademas control de acceso corporativo, gobernanza de datos y soporte documental contextualizado.

9) Summary de la app

Que resuelve

Permite a operarios de centros de control consultar datos operativos en tiempo real, generar reportes y acceder a manuales, sin conocimiento SQL ni dependencia de especialistas.

Como lo hace

Combina Streamlit, agentes IA (LangChain + Azure OpenAI), validacion SQL, RAG con n8n, y exportacion de resultados reutilizables.

Valor final

Reduce friccion operativa, acelera la toma de decisiones en tiempo real y aporta trazabilidad tecnica con gobierno corporativo integrado.

10) Posibles preguntas tecnicas

1. Como evitas que el LLM genere SQL inseguro?

Con una cadena de seguridad: guardrails en prompts, validacion previa del SQL y ejecucion en modo solo lectura (solo SELECT), sin permisos de escritura.

2. Cual fue tu rol en este proyecto?

El proyecto es una Proof of Concept; no lo desarrolle desde cero. Mi aportacion fue mejorar diseno y funcionalidades IA: ajuste de prompts, correccion de errores y generacion de datos sinteticos con el mismo schema de la BBDD para testing en desarrollo.
